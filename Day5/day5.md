# Day 5 – Context Engineering

## Prompt A (Without Context)
Create a 30-day learning roadmap.

Include:
- Weekly milestones
- Daily tasks
- Resources
- Projects
- Final outcome

Make it practical and beginner-friendly.

### Observation
- The roadmap was generic.
- It focused on basic programming concepts.
- It was suitable for any beginner.
- No personalization based on my goals or skills.

## Prompt B (With Context)
Create a 30-day learning roadmap.

Context:
- Current Situation: [Student/Professional/Freelancer]
- Current Skills: [Add Skills]
- Goal: [Target Goal]
- Available Time: [Hours per Day]
- Experience Level: [Beginner/Intermediate]
- Preferred Learning Style: [Videos/Projects/Reading]

Include:
- Weekly milestones
- Daily tasks
- Resources
- Projects
- Final outcome

Make it practical and beginner-friendly.

### Observation
- The roadmap was customized for a B.Tech IT student.
- It focused on MERN Stack and interview preparation.
- Daily tasks matched my available learning time.
- Resources and projects were aligned with my career goal.

# Comparison

## 1. Which roadmap feels more personalized?

Prompt B because it considered my current skills, learning time, experience level, and career goal.

## 2. Which roadmap would you actually follow?

Prompt B because it is practical, realistic, and directly supports my goal of becoming a Full Stack Developer.

## 3. What role did context play in improving the result?

Providing context helped Claude generate a roadmap tailored to my needs instead of giving a generic learning plan. The recommendations, projects, and resources became more relevant and actionable.

---

# Key Learnings

- Better context leads to better AI responses.
- AI can generate personalized learning plans when given sufficient information.
- Context Engineering improves relevance, accuracy, and usability.
- Small details like available time and experience level significantly affect the quality of the roadmap.
